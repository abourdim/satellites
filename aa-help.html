⚙️ Tips⚙️ Astuces⚙️ نصائح

Audio experiences (Doppler Demo, Radio from Space, Sky Music) — click/tap to enable audio. Browsers require user interaction before playing sound.

Canvas simulations (Kessler, Orbit Decay, Re-entry) — work best on desktop. On mobile, they still function but the controls are more compact.

Scrolling experiences (Pale Blue Dot, Zoom Out) — use mouse wheel or two-finger swipe on mobile. Scroll slowly for the best effect.

Portal navigation — each file has a "back to portal" link or you can use browser back. Files are named aa-[topic].html for easy identification.

🔧 Troubleshooting🔧 Dépannage🔧 استكشاف الأخطاء

Fonts look different
Google Fonts need internet on first load. After caching, they work offline. Without them, the system falls back to default sans-serif — everything still works.
No audio in Doppler/Radio
Click or tap the page first. Browsers block autoplay audio until user interaction. Look for "Listen" or "Play" buttons.
Canvas looks blurry
Most simulations handle high-DPI displays. If blurry, try zooming to 100% in browser (Ctrl/Cmd + 0).
Links between files don't work
All files must be in the same folder. Don't rename files. The portal links use relative paths like aa-kessler.html.
File won't open
Double-click the .html file, or right-click → Open With → your browser. Works with any modern browser.
